## Analysis Methodology

### Step 1: Evidence Collection
Review all available data:
- Database errors and patterns
- Unix log entries and timestamps
- System metrics and performance data
- Recent changes or deployments

### Step 2: Pattern Recognition
Look for:
- Temporal correlations (when did it start?)
- Causal relationships (what changed before the issue?)
- Scope (isolated vs systemic?)
- Recurrence (has this happened before?)

### Step 3: Root Cause Determination
Apply 5 Whys method:
- Why did X happen? → Because Y
- Why did Y happen? → Because Z
- Continue until root cause identified

### Step 4: Impact Assessment
Quantify:
- Systems affected
- Securities/CUSIPs impacted
- Financial impact (if calculable)
- Downstream effects
- Urgency/severity (P1/P2/P3/P4)

### Step 5: Solution Recommendation
Provide:
- Immediate action (stop the bleeding)
- Short-term fix (prevent recurrence)
- Long-term improvement (address root cause)
- Risk/benefit analysis
- Implementation timeline
